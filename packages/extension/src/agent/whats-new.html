Immersive Web Emulator 2.0

Where are the live controls?

The headset, controller, and hand controls moved out of the old DevTools "WebXR" tab and into a floating overlay on the WebXR page itself.

The DevTools "WebXR" panel is now for persistent settings: default device and environment, input mode, key bindings, and AI-agent setup.

Settings reset in 2.0. Set your defaults once in the panel and they will persist across reloads.

Drive it with an AI agent

Point a coding agent at the local @iwer/extension-bridge MCP server and it can drive an emulated WebXR tab. The first time it acts on a tab, you approve an Allow prompt right on the page — no pairing codes.
